# Tech/AI word vetting — keep only the genuinely brandable

This is an auto-generated tech/AI/startup word list. Keep only the ones that are
genuinely strong as a tech product / startup / domain name. Be selective — keep
roughly the best third to half. We do NOT want weak or junky entries.

KEEP if the word is:
- a real or well-established tech/AI/dev/crypto/cloud noun, AND
- clean, pronounceable, and brandable (a startup would proudly use it), AND
- the kind of name a domain investor would see real value in.

Good KEEP examples: agent, prompt, vector, token, kernel, cipher, beacon, forge,
prism, vertex, cascade, daemon, webhook, runtime, pipeline, schema, proxy,
gateway, ledger, wallet, oracle, vault, cluster, node, mesh, cache, sandbox,
payload, lambda, neuron, synapse, signal, spark, nimbus, atlas, relay, pulse.

CUT:
- units / measures (watt, ampere, hertz, joule)
- proper nouns / star / mythology names used as filler (altair, antares, apollo,
  orion) UNLESS genuinely brandable and generic
- dull abstract -ion/-ity/-ment terms (abstraction, activation, allocation)
- obscure jargon, internal-tool words, or anything most people wouldn't know
- acronyms, adjectives, verbs-only, ugly/unpronounceable coinages, gibberish
- anything that just isn't a strong brandable noun

When torn, CUT — we want a tight, high-quality tech set.

## Output rules

For each assigned batch file: read it (a JSON array), apply this bar, and write
the kept words as a JSON array to the matching output path. Only output words
that appear EXACTLY in the input (lowercase). No new words. Valid JSON only.
